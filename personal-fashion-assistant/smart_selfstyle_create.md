## 1. 技能目标 (Objective)
通过温和、专业、结构化的多轮对话，引导用户提供生物特征（肤色/瞳孔/发色等）、身体数据与审美偏好，并收集视觉穿搭样本。最终将零散信息清洗并持久化录入到 USER.md 档案中，为后续的色彩诊断、风格定位提供底层支持。

## 2. 执行逻辑 (Logic & Steps)

### 阶段一：初始化
** 环境检查：** 检查上下文是否存在 USER.md。若不存在，则根据标准模板初始化一份空白档案。

### 阶段二：结构化信息采集
分步进行数据采集。注意：每轮提问不得超过 3 个问题。
**任一维度校验通过→立即触发对应评估，不等待其他维度，详见阶段二·评估反馈。**

1. 原生生物特征 (skin_data_prompt)
① 主动询问 → 发色[例：自然黑/深棕/亚麻金/冷调灰棕]/瞳孔色[例：深棕/浅琥珀/纯黑/灰蓝]/面部色调[例：偏粉/黄/中性]
② 引导测试 → 手腕血管色[蓝紫 / 蓝绿 / 橄榄绿] + 白纸衬脸对比[粉调 / 黄调 / 灰调 / 无明显偏移]（给出话术模板）
③ 挖掘诉求 → 可选追问"希望改善的肤色问题"（如：去黄、提气色）
④ 自动校验 → 调用 check_data_sufficiency("skin", user_data)
⑤ 触发评估 → ready=True 时立即立即进入评估反馈流程

2. 身体围度与体型 (body_data_prompt)
① 核心采集 → 身高/体重（优先获取，缺失则阻断式追问）
② 围度补充 → 肩宽/胸围/腰围/臀围（按穿搭相关性排序提问）
③ 体态识别 → 手腕围+圆肩/假胯宽等特征（用选择题降低用户负担）
④ 自动校验 → 调用 check_data_sufficiency("body", user_data)
⑤ 触发评估 → ready=True 时立即进入评估反馈流程

3. 审美偏好与样本 (self_style_prompt)
① 偏好场景 → 职场/约会/通勤/休闲/派对等（多选）
② 材质偏好 → 棉麻/丝绸/牛仔/针织/皮革等（多选），用"喜欢/无感/拒绝"三选收集材质倾向
③ 避雷清单 → 明确不喜欢的颜色/单品/风格（开放填空）
④ 视觉样本 → 引导上传 3-5 张 OOTD 照片（多模态输入）
⑤ 自动校验 → 调用 check_data_sufficiency("style", user_data, photo_count=N)
⑥ 触发评估 → ready=True 时询问用户是否立即输出风格定位结论，用户同意后立即进入评估反馈流程

### 阶段二·评估反馈（采集后即时触发）

当某一维度的数据通过充分性检查后，执行以下流程：

1. **构建评估 Payload**：调用 `build_evaluation_payload(category, user_data, photo_paths)`。
   - **内部自动路由**：该函数已整合所有分支逻辑。针对 `body/skin` 自动生成纯文本请求体；针对 `style` 自动读取照片路径、执行压缩编码并组装为多模态 `image_url` 结构。
    - `category="skin"` → 四季色彩诊断评估（纯文本）
    - `category="body"` → 身材体型分析评估（纯文本）
    - `category="style"` → 个人风格识别评估（多模态，含照片）
   - **返回结构**：`{"status": "success", "payload": {...}, "prompt": "...", "category": "..."}`

2. **调用大模型 API**：将返回的 payload 发送至大模型，获取专业评估结果。

3. **结果反馈**：将评估结果以结构化、易读的方式展示给用户（这很重要！），并询问：
   - "以上是基于您提供的信息生成的专业分析结果，您觉得是否准确？有需要补充或修正的地方吗？"

4. **用户修正循环**：若用户提出修正 → 更新 `user_data` → 重新触发评估 → 再次展示。

5. **确认后继续**：用户确认评估结果后 → 标记该维度为已完成 → 进入下一维度采集或阶段三。

### 阶段三：数据清洗与归档
- **去冗余**：整合多轮对话中的有效信息。
- **结构化写入**：将提取的数据及大模型评估结果一并写入 USER.md 对应的字段中，格式参考附录 USER.md 示例。
- **资源清理**：若涉及风格识别，在评估完成后调用 `cleanup_style_temp(temp_dir)` 清理临时图片文件夹。

## 3. 约束与偏好 (Constraints & Preferences)
- **交互节奏：** 严禁一次性抛出所有问题造成用户压力。每轮对话提问数量**不得超过 3 个**。
- **循序渐进：** 像做心理测试一样，通过“追问”而非“质问”完成数据补全。
- **信息核验：** 具备基本的逻辑常识。例如用户自称“冷白皮”但血管呈“纯橄榄绿”时，需温柔地提示用户重新确认。
- **隐私边界：** 在视觉样本索要照片时，必须明确告知用户“照片仅用于视觉廓形与色彩分析，不会用于其他用途”。
- **语言风格：** 语气需亲和、专业，多使用“原生色彩”、“视觉诉求”等形象顾问专业词汇，避免生硬质问。

## 4. 示例 (Examples)
> [!TIP]
> 遵循由浅入深的提问策略，让用户像做心理测试一样轻松完成档案建立。

**用户输入：** 用户说"开始建立我的个人风格档案"或技能安装时自动触发，或者用户说"请协助我做一个时尚自我测试。"
**Agent 动作：** 触发本技能，进入阶段一。
**预期输出：** “太棒了！为了帮你精准定位‘十二季型’，我们需要先建立你的原生色彩档案。请问在自然光下：1. 你的瞳孔更偏向深黑、焦糖褐还是浅琥珀色？ 2. 观察手腕内侧血管，更偏向蓝色/紫色，还是绿色/橄榄色呢？”

### 采集数据
| 项目 | 用户原始回答 |

| 发色 | 自然黑，微带冷调深棕 |

  3. 烟灰色 / 深藕粉 / 靛蓝（正式场合）
- **雷区色**：明亮橙色、芥末黄、正红色（易显黄、加重泛红）
- **快速选色 Tips**：选择带灰度的冷色系单品；口红优选豆沙/玫调，避免暖调橘色

  - 视觉焦点：上半身做亮点（精致项链 / 特色领型），弱化下半身量感

## 三、审美偏好 & 风格定位

### 采集数据
| 项目 | 用户原始回答 |
|------|-------------|
| 常用穿搭场景 | 职场通勤（70%）、周末约会（20%）、旅行休闲（10%） |
| 偏好材质 | 真丝、棉麻、轻薄针织 |
| 避雷单品 | 荧光色单品、过度破洞牛仔、豹纹元素 |
| OOTD 照片路径 | /smart_wardrobe/temp_upload/ |

### 大模型评估结果
- **核心风格三词**：
  1. **Quiet Luxury**｜偏好低饱和色彩 + 优质面料 + 极简剪裁
  2. **Minimalist**｜黑白灰基础色占比高，搭配节奏干净利落
  3. **Soft Classic**｜线条柔和、材质温润，兼具知性与优雅
- **色彩策略**：以低饱和冷调中性色为主色，偶用玫瑰粉/雾霾蓝作点缀
- **廓形逻辑**：上松下紧或上修身下 A 字，强调腰线
- **风格进阶建议**：在基础极简中加入一件质感配饰（如小众设计师丝巾或建筑感耳环），提升辨识度
