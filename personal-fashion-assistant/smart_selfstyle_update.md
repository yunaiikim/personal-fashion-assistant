## 1. 技能目标 (Objective)
提供用户查询和修改 USER.md 中个人时尚数据的能力。当用户需要更新身体数据、肤色信息或风格偏好时，先展示当前档案内容，再根据用户指令精准修改对应字段，并重新触发大模型评估以确保分析结果与最新数据同步。

## 2. 执行逻辑 (Logic & Steps)

### 阶段一：意图识别与档案读取
** 环境检查：** 检查上下文是否存在 USER.md。若不存在，提示用户先执行 `smart_selfstyle_create` 建立档案。

** 意图路由：** 根据用户输入判断操作类型：

    "请问你想做什么？（回复数字即可）：
    1️⃣ 【查看档案】：查看我的个人时尚数据
    2️⃣ 【修改数据】：更新某项个人信息（如体重变化、发色变化等）
    3️⃣ 【重新评估】：基于现有数据重新获取大模型分析"

### 阶段二：数据查询与展示

根据用户意图，读取 USER.md 并按维度展示现有数据。

1. 全量查看
读取 USER.md 全部个人时尚相关字段，以结构化卡片形式分维度展示：
   - 🎨 **肤色档案**：发色、瞳孔色、面部色调、血管测试、白纸测试、季型诊断结果
   - 📐 **身材档案**：身高、体重、各项围度、体型判定、骨架量感
   - 👗 **风格档案**：穿搭场景、材质偏好、避雷单品、风格关键词

2. 按维度查看
用户指定某一维度（肤色/身材/风格）时，仅展示该维度的详细数据。

### 阶段三：定向修改

当用户提出修改请求时，执行以下流程：

1. **展示现有值**：先显示用户要修改的字段当前值，确保用户了解原始数据。
   - "您当前的 [字段名] 记录为：[当前值]，请告诉我新的数据。"

2. **接收新值**：接收用户提供的新数据，进行基本合理性校验。
   - 数值类字段（身高/体重/围度）：检查数值是否在合理范围内。
   - 枚举类字段（血管色/白纸测试）：检查是否为有效选项。
   - 若检测到可能的逻辑矛盾（如身高从 165cm 改为 265cm），温柔提醒用户确认。

3. **确认修改**：向用户展示修改前后的对比，等待确认。

        修改预览：
        ┌──────────┬──────────┬──────────┐
        │ 字段      │ 修改前    │ 修改后    │
        ├──────────┼──────────┼──────────┤
        │ 体重      │ 50kg     │ 55kg     │
        └──────────┴──────────┴──────────┘
        确认修改？（回复"确认"或"取消"）

4. **写入 USER.md**：用户确认后，更新 USER.md 中对应字段。

### 阶段四：重新评估（可选）

当修改涉及关键字段时，建议用户重新触发大模型评估：

1. **判断是否需要重新评估**：
   - 肤色相关字段变更（发色/瞳孔色/血管测试等）→ 建议重新评估季型诊断
   - 身材相关字段变更（体重/围度等）→ 建议重新评估体型分析
   - 风格相关字段变更（场景/材质/新 OOTD 照片）→ 建议重新评估风格识别

2. **触发评估**：调用 `build_evaluation_payload(category, user_data, photo_paths)` 构建 payload。

3. **调用大模型 API**：AI Agent 将 payload 发送至多模态大模型，获取更新后的评估结果。

4. **结果反馈与对比**：展示新旧评估结果的差异，帮助用户理解数据变化带来的影响。
   - "体重变化后，您的体型判定从 [旧结果] 更新为 [新结果]，服装建议也相应调整。"

5. **确认并归档**：用户确认后，将新的评估结果写入 USER.md。

## 3. 约束与偏好 (Constraints & Preferences)
- **档案前置：** USER.md 必须已存在，否则引导用户先执行 `smart_selfstyle_create` 创建档案。
- **先展后改：** 所有修改操作必须先展示当前值，严禁直接覆盖。
- **修改确认：** 任何字段的修改都必须经过用户确认后才能写入 USER.md。
- **合理性校验：** 对数值类字段进行范围检查，对矛盾数据温柔提醒。
- **语言风格：** 与 `smart_selfstyle_create` 保持一致——亲和、专业，使用形象顾问专业词汇。

## 4. 示例 (Examples)
> [!TIP]
> 修改操作遵循"先看 → 再改 → 确认 → 可选重评"的四步流程。

**示例一（查看档案）：**
**用户输入：** "看看我的肤色数据。"
**Agent 动作：** 读取 USER.md 肤色相关字段，结构化展示。
**预期输出：** "🎨 您的肤色档案如下：
• 发色：自然黑
• 瞳孔色：深棕带金边
• 面部色调：暖黄调
• 血管测试：蓝绿
• 季型诊断：柔夏型（Soft Summer）
有需要修改的地方吗？"

**示例二（修改数据）：**
**用户输入：** "我的体重变成55kg了。"
**Agent 动作：** 读取当前体重 → 展示对比 → 等待确认 → 写入 → 建议重新评估。
**预期输出：** "好的，您的体重将从 50kg 更新为 55kg。确认修改吗？修改后建议重新评估身材分析，以便更新体型判定和服装建议。"
