## 1. 技能目标 (Objective)
提供用户修改或删除衣橱内指定衣物记录的能力。在用户通过 `clothing_search` 确认了目标衣物后，根据用户的指令，对 `inventory.md` 中的记录进行更新（如修改颜色、季节、标签等）或彻底删除（出库），必要时处理对应的图片文件，并记录出库日志。

## 2. 执行逻辑 (Logic & Steps)

### 阶段一：意图识别与目标确认
1. **前置条件验证**：确保用户已经明确指定了要操作的衣物（通常应该是在执行 `clothing_search` 后，用户已经确认了某个具体的衣物 ID）。
2. **提取操作意图**：从用户的指令中提取是“修改（Update）”还是“删除（Delete/移除/出库）”。
   - 修改示例："把这件衣服的风格改成少女风"，"给这件大衣加上'通勤'的标签"。
   - 删除示例："把这条破洞牛仔裤从衣橱里删掉"，"这件衣服我扔了，出库吧"。

### 阶段二：执行修改 (Update逻辑)
1. **定位记录**：根据确认的衣物 ID，在 `inventory.md` 的表格中找到对应的行记录。
2. **更新字段**：根据用户的要求，修改对应属性列的值。
3. **保存并反馈**：将修改后的记录保存回 `inventory.md`，并向用户展示修改后的信息。
   - "已为您更新该衣物的记录。新的属性如下：..."

### 阶段三：执行删除/出库 (Delete逻辑)
1. **二次确认**（可选但推荐）：若用户指示删除，可以进行一次安全确认："您确定要将这件衣物（ID: XXXXXXXXXXXX）从衣橱中彻底移除吗？"
2. **移除记录**：在 `inventory.md` 中删除该 ID 对应的整行记录。
3. **处理关联文件**（可选策略，视具体实现定）：
   - 将 `wardrobe_data/` 目录下对应的图片文件（`[ID].jpg/png`）移动到类似 `archive_deleted/` 的回收站目录，或者直接删除。
4. **记录出库日志**：在相关的出库日志文件（如 `outbound_log.md`）中记录删除操作（包含ID、删除时间、原因等）。
5. **保存并反馈**：保存 `inventory.md` 的变更，并告知用户已成功移除。
   - "好的，已将该衣物（ID: XXXXXXXXXXXX）从您的电子衣橱中移除，并记录出库。"

## 3. 约束与偏好 (Constraints & Preferences)
- **准确定位**：所有的修改和删除操作必须基于准确的衣物 ID，避免误操作。如果用户指令模糊，必须先引导至 `clothing_search` 确认具体的衣物。
- **数据完整性**：修改记录时，应保持 `inventory.md` Markdown 表格的格式完整对齐，不能破坏表格语法。
- **不可逆操作提示**：删除出库操作通常是不可逆的（或者需要从日志恢复），建议在执行删除前向用户确认。

## 4. 示例 (Examples)

**示例一（修改记录）：**
*(前置上下文：用户刚通过搜索确认了 ID 为 202310150001 的黑色大衣)*
**用户输入：** "帮我把这件大衣的适合场景加上'休闲'。"
**Agent 动作：**
1. 读取 `inventory.md`，找到 ID `202310150001` 的记录。
2. 将其“适合的场景”字段更新，追加“休闲”。
3. 保存 `inventory.md`。
**预期输出：** "已为您更新记录！这件黑色大衣的使用场景现在包含了'休闲'。"

**示例二（删除/出库记录）：**
*(前置上下文：用户刚通过搜索确认了 ID 为 202305100003 的破洞牛仔裤)*
**用户输入：** "这件我上周扔了，把它删掉吧。"
**Agent 动作：**
1. 读取 `inventory.md`，删除 ID `202305100003` 的整行记录。
2. 移动/删除 `wardrobe_data/202305100003.png`。
3. 记录出库日志。
**预期输出：** "处理完毕！这条破洞牛仔裤（ID: 202305100003）已从您的衣橱库中移除，出库记录已登记。"
